# 上下文捕获 Hook

## 1. 定位

本 Hook 在 Agent 会话结束时提出上下文更新建议，用于持续改进 `docs/PROJ_CONTEXT.md`、`AGENTS.md`、子目录上下文文件和排除路径的准确性和完整性。它不自动写入任何长期上下文文件，所有建议需由 hicode Owner 人工确认后归并。

规则依据：`skills/init/coding_rules.md`、`target-project:docs/rules/`。
适用动作：Agent 会话结束、会话停止。

## 2. 触发点

| 项 | 内容 |
|---|---|
| Hook ID | `context-capture-hook` |
| 建议适配事件 | `after_session_end`、`stop` |
| 规则依据 | `skills/init/coding_rules.md`、`target-project:docs/rules/` |
| 默认模式 | `advisory` |

## 3. 输入材料

Hook 执行前需要从本次会话收集：

1. **已读取文件列表**：本次会话中 Agent 实际读取的文件路径。
2. **已修改文件列表**：本次会话中 Agent 实际修改的文件路径。
3. **已运行命令列表**：本次会话中 Agent 实际执行的命令及结果摘要。
4. **搜索模式记录**：本次会话中使用的搜索关键词、路径和命中范围。
5. **发现项**：本次会话中发现的新模块、新命令、新排除路径、新风险或新依赖。

不得要求真实客户敏感信息、未脱敏生产数据、生产配置、生产日志原文、密钥或生产凭证。

## 4. Advisory 输出

提醒型输出为上下文更新建议列表，包含以下类别：

1. **建议新增的 PROJ_CONTEXT 条目**：新发现的模块、接口、表、配置、依赖或风险点。
2. **建议修改的 PROJ_CONTEXT 条目**：已变更的命令、过期的路径、错误的信息。
3. **建议废弃的 PROJ_CONTEXT 条目**：已删除的模块、不再使用的命令或配置。
4. **建议新增/修改的排除路径**：本次会话中发现应排除但未排除的目录。
5. **建议新增/修改的子目录上下文文件**：符合子目录上下文创建条件但尚未创建的模块。
6. **建议新增/修改的局部命令**：本次会话中使用但未记录在命令矩阵中的命令。

每条建议应包含建议类型（新增/修改/废弃）、目标文件、目标章节、建议内容和依据。

## 5. Blocking 条件

无。本 Hook 为纯 advisory-only，不阻断任何流程。

## 6. 禁止动作

1. 不自动写入任何长期上下文文件（包括 `PROJ_CONTEXT.md`、`AGENTS.md`、`DOMAIN_KNOWLEDGE.md`、子目录上下文文件等）。
2. 不自动修改 `AGENTS.md` 或 `PROJ_CONTEXT.md`。
3. 不自动更新排除路径配置。
4. 不自动创建或修改子目录上下文文件。
5. 不自动提交、推送、合并或发布。
6. 不连接生产环境或读取生产数据。

## 7. 审计证据

建议输出归档到会话输出或临时文件，由 hicode Owner 人工确认后归并。审计证据包括：

1. 会话已读取文件清单。
2. 会话已修改文件清单。
3. 会话搜索模式和命中范围记录。
4. 上下文更新建议列表（含建议类型、目标、内容和依据）。
5. 人工确认记录（确认人、确认时间、采纳或拒绝说明）。
