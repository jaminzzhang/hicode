## 2. 状态规则

| 状态 | 含义 | 使用规则 |
|---|---|---|
| 未开始 | 工作包尚未启动 | 只有前置依赖或优先级确认后才启动 |
| 进行中 | Agent 正在处理该工作包 | 开始实施时更新为该状态 |
| 阻塞 | 缺少输入、确认或外部条件 | 必须写明阻塞原因和解除条件 |
| 待验收 | 产出已完成，等待项目负责人确认 | Agent 完成工作包后使用该状态 |
| 已完成 | 项目负责人已确认产出可接受 | 只有确认后才能使用该状态 |
| 暂缓 | 主动推迟，当前不是阻塞 | 必须写明暂缓原因和恢复条件 |
