# RELEASE_GUIDE

## 1. 文档定位

本文档记录目标项目发布前检查、发布风险识别、回滚和上线后观察要求。

- 维护对象：发布负责人、研发负责人、测试负责人
- 更新时机：发布流程变化、发布事故复盘、环境或配置规则变化后
- 主要使用者：发布前检查、发布准入门禁 Agent
- 当前版本：V1 初版规则，Agent 不自动发布，只提供检查结论和风险建议

## 2. 发布前输入

发布前检查应尽量准备：

1. 神兵版本链接或版本标识。
2. 发布需求清单。
3. 代码分支。
4. Commit / MR 清单。
5. 制品版本。
6. 环境配置。
7. SQL 脚本。
8. 测试报告。
9. 缺陷关闭情况。
10. 发布说明。
11. 回滚方案。
12. `docs/RELEASE_GUIDE.md`。

## 3. 发布结论

| 结论 | 使用条件 | 下一步 |
|---|---|---|
| 建议发布 | 发布范围、分支、制品、测试、缺陷、SQL、配置、验证点和回滚均满足要求 | 进入人工发布审批 |
| 有条件发布 | 存在非阻断风险，责任人已确认补救措施 | 补充说明后进入人工发布审批 |
| 不建议发布 | 存在未关闭 P0/P1 缺陷、测试未通过、SQL/配置/回滚缺失或敏感信息风险 | 修复后重新检查 |

## 5. 发布前检查项

| 检查项 | 要求 | 证据来源 | 未满足风险 |
|---|---|---|---|
| 金融核心系统风险基线 | 保险核心业务逻辑、金额、交易一致性、状态流转、幂等、权限、审计、隐私、监管、生产变更和回滚风险已有检查结论 | 需求上下文、测试报告、Review 报告、发布检查报告 | 核心业务错误或生产事故 |
| 需求清单 | 发布需求清单完整，无未纳入需求 | 版本链接、需求清单 | 漏发、错发 |
| 分支和制品 | 分支正确，制品版本与发布申请一致 | 分支信息、制品记录 | 错发版本 |
| Commit / MR | 无漏合、错合、多合，MR 已 Review | Commit / MR 清单 | 代码不一致 |
| SQL | SQL 完整，影响范围明确，有回滚 SQL 或回滚说明 | SQL 清单 | 数据错误、无法回滚 |
| 配置 | 各环境配置齐备，生产配置已人工确认 | 配置清单 | 配置遗漏、生产故障 |
| 接口 | 接口兼容性和调用方影响已确认 | 接口文档、影响范围 | 兼容性故障 |
| 测试 | 测试通过，核心场景覆盖 | 测试报告、核心场景测试报告 | 漏测 |
| 缺陷 | 无未关闭 P0/P1 缺陷，高风险缺陷有处理结论 | 缺陷清单 | 带缺陷上线 |
| 敏感信息 | 无敏感信息泄露或密钥提交 | 扫描结果、提交检查报告 | 安全事件 |
| 验证点 | 生产验证点明确，责任人明确 | 发布说明 | 上线后无法确认 |
| 回滚 | 回滚方案可执行，触发条件明确 | 回滚方案 | 无法止损 |

## 6. SQL 检查

| 检查项 | 要求 | 状态 |
|---|---|---|
| SQL 清单 | 每个脚本有用途、执行顺序和影响范围 | 待确认 |
| 回滚 SQL | 有回滚 SQL 或说明无法回滚的原因和补救方案 | 待确认 |
| 数据影响 | 明确影响表、字段、数据量和业务影响 | 待确认 |
| 执行环境 | 明确测试、预发、生产执行差异 | 待确认 |
| 审批记录 | 有人工确认记录 | 待确认 |

## 7. 配置检查

| 检查项 | 要求 | 状态 |
|---|---|---|
| 配置清单 | 明确新增、修改、删除配置项 | 待确认 |
| 环境差异 | 明确各环境配置差异 | 待确认 |
| 生产确认 | 生产配置由负责人确认，Agent 不读取或修改生产配置 | 待确认 |
| 回退方案 | 配置回退路径明确 | 待确认 |
| 敏感配置 | 密钥、Token、连接串不出现在报告或示例中 | 待确认 |

## 8. 测试和缺陷检查

| 检查项 | 要求 | 状态 |
|---|---|---|
| 保险核心业务逻辑 | 投保、批改、退保、核保、理赔前置、收付费、责任判断等核心逻辑已有测试或负责人确认 | 待确认 |
| 单元测试 | 核心逻辑测试通过或有明确例外说明 | 待确认 |
| 核心场景 | 投保、批改、退保、收付费、渠道等相关核心场景已覆盖 | 待确认 |
| 回归测试 | 影响范围内回归完成 | 待确认 |
| 缺陷关闭 | 无未关闭 P0/P1 缺陷 | 待确认 |
| 历史缺陷 | 相似历史缺陷已回归 | 待确认 |

## 13. 禁止事项

1. Agent 不自动发布。
2. Agent 不修改生产配置。
3. Agent 不操作生产环境。
4. Agent 不输出生产账号、密钥或客户敏感信息。
5. Agent 不替代发布负责人做最终发布决策。

## 14. 发布检查报告结构

发布前检查输出应包含：

1. 发布结论。
2. 发布范围。
3. 需求清单核对。
4. 分支和制品核对。
5. Commit / MR 核对。
6. SQL 检查。
7. 配置检查。
8. 测试报告检查。
9. 缺陷关闭情况。
10. 生产验证点。
11. 回滚方案。
12. 发布风险。
13. 上线后观察建议。
