# Java Review Rules

## 1. 定位

本文档记录 Java、Spring、事务、异常、日志、分层和外部调用相关的代码审查细则。

- 上级总则：`docs/REVIEW_RULES.md`
- 触发条件：diff 涉及 Java 代码、Spring 配置、服务层、DAO、事务、异常、日志或外部调用
- 使用边界：只作为代码审查细则，不替代 `docs/CODING_RULES.md` 和 `docs/TESTING_GUIDE.md`

## 2. 分层与职责

| 检查项 | 风险 | 常见等级 |
|---|---|---|
| Controller / Service / DAO 职责混乱 | 业务逻辑不可审查、事务边界不清 | P1/P2 |
| 跨层直接调用或绕过统一服务 | 权限、审计、事务规则失效 | P1 |
| 公共组件引入业务分支 | 影响范围扩大 | P1/P2 |
| 领域对象、DTO、持久化对象混用 | 字段含义和敏感信息边界不清 | P1/P2 |

## 3. 事务与一致性

1. 核心交易必须明确事务边界。
2. 不得在长事务中执行不可控远程调用。
3. 跨系统调用必须说明补偿、重试或最终一致性策略。
4. 批改、退保、收付费、状态更新等操作必须防止部分成功导致数据不一致。
5. 重复请求、重复回调和并发更新必须有幂等或锁控制。

## 4. 异常处理

| 检查项 | 未满足时建议等级 |
|---|---|
| 吞异常或只打印日志不处理 | P1 |
| 异常类型不能表达业务含义 | P2 |
| 外部调用失败未区分超时、失败、重复响应 | P1 |
| 核心流程异常后没有回滚、补偿或状态标记 | P0/P1 |
| 错误信息暴露客户敏感信息或内部实现 | P0/P1 |

## 5. 日志与审计

1. 核心交易、状态变更、审批动作和外部调用应有必要日志或审计记录。
2. 日志必须脱敏，不得输出证件号、手机号、银行卡号、保单号、客户号、Token、Cookie、Session 或连接串原文。
3. 日志应包含可追踪的请求编号、业务编号或脱敏后的关联标识。
4. 不得用日志替代业务状态、补偿记录或审计字段。

## 6. 外部调用

1. 必须检查超时、重试、熔断、降级和失败处理。
2. 不得默认外部系统一定成功。
3. 回调类接口必须校验幂等、签名、来源和状态合法性。
4. 涉及客户信息、支付、保单、批单或理赔前置调用时，必须标注 P0/P1 风险判断。

## 7. 测试关注点

1. 核心 Java 服务应覆盖正常、边界、异常和重复请求场景。
2. 单元测试不得访问真实数据库、真实外部系统或生产配置。
3. Mock 必须覆盖成功、失败、超时、重复响应和非法状态。
4. 对金额、日期、状态、权限和幂等逻辑必须有断言。
