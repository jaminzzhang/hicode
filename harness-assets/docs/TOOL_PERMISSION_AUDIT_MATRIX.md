# TOOL_PERMISSION_AUDIT_MATRIX

## 1. 文档定位

本文档定义 V1 Harness Prompt、Skill 和门禁在目标项目中的工具权限边界、禁止操作和审计证据口径。

- 维护对象：AI 工程化小组、研发负责人、测试负责人、安全负责人
- 主要使用者：开发 Agent、试点研发人员、Reviewer、发布负责人
- 适用范围：需求评审、编码计划、TDD、辅助编码、代码审查、提交检查、核心场景测试、发布检查和节点门禁

本文档不定义目标项目用户 RBAC、组织审批流、生产发布权限模型或人员职责分配。

## 2. 权限等级

| 等级 | 名称 | V1 使用口径 |
|---|---|---|
| L0 | 只读分析 | 读取脱敏材料、上下文、规范、diff、报告或检查结果，输出分析结论 |
| L1 | 生成建议 | 输出计划、报告、测试设计、补丁草案、MR 描述草稿、上下文更新建议或门禁建议 |
| L2 | 本地修改 | 在目标项目工作区内修改代码、测试或必要文档；V1 默认只授予辅助编码场景 |
| L3 | 受限命令 | 执行本地非生产测试、构建、静态检查、格式检查或敏感信息扫描，并记录结果 |
| L4 | 平台接入 | 接入 CI/CD、发布平台、流程平台或代码托管平台执行动作；V1 不作为默认能力 |

V1 默认控制在 L0-L2。L3 只能在目标项目已有脚本、文档、工具配置或用户明确确认时按条件开放。L4 在 V1 中写为禁止或转既有人工/平台流程。

## 4. 全局禁止操作

所有场景均禁止 Agent 执行：

1. 读取或输出生产账号、密码、Token、Cookie、Session、连接串、生产 IP 或内部密钥。
2. 读取 `.env`、密钥文件、生产配置文件或生产凭证。
3. 处理未脱敏真实客户数据或未脱敏生产数据。
4. 删除或覆盖未确认文件。
5. 执行数据库变更、生产连接、生产 SQL、生产日志查询或生产接口调用。
6. 自动提交、自动推送、自动合并、自动发布、自动回滚或自动修改生产配置。
7. 用删除测试、降低断言、跳过 Review、隐藏 P0/P1 风险或弱化敏感信息风险推动通过。

需要上述动作时，Agent 只能输出风险、证据缺口、建议确认人和转既有人工流程或平台流程的建议。

## 5. 受限命令口径

受限命令必须同时满足：

1. 命令来自目标项目已有脚本、文档、工具配置或用户明确确认。
2. 命令运行在本地非生产环境。
3. 命令不连接生产、不读取密钥、不处理未脱敏生产数据。
4. 命令不执行数据库变更、发布、回滚、提交、推送、合并或删除未确认文件。
5. 命令执行结果必须记录命令、范围、授权来源、是否执行、结果、失败摘要或未执行原因。

V1 允许的受限命令类型仅包括本地测试、构建、静态检查、格式检查、覆盖率、敏感信息扫描和只读 Git 查询。

## 6. 审计证据口径

每次 Prompt、Skill 或门禁输出应尽量保留可归档证据：

| 证据类型 | 内容 |
|---|---|
| 输入材料清单 | 需求、上下文、diff、报告、测试结果、CI 摘要、人工确认记录等来源 |
| 检查范围 | 需求编号、分支、Commit/MR、版本、制品、发布申请或节点范围 |
| 结论依据 | 结论、风险等级、关键证据、缺口和不适用说明 |
| 受限命令记录 | 命令、范围、授权来源、结果、失败摘要或未执行原因 |
| 阻断建议 | P0/P1、敏感信息、生产越权、关键证据缺失等问题及解除条件 |
| 普通风险提示 | P2/P3、材料补充、验证增强、负责人风险接受条件 |
| 上下文更新建议 | 建议更新位置、类型、原因和待确认事项 |

审计证据用于说明 Agent 建议如何形成，不代表负责人审批已完成。

## 7. 维护规则

1. 新增 Prompt、Skill、门禁或受限命令能力时，必须检查本文档是否需要更新。
2. 若某资产声明可修改文件或执行命令，必须明确权限等级、允许范围、禁止事项和审计证据。
3. 若某资产涉及生产验证、SQL、配置、回滚或客户敏感信息，必须显式写明 Agent 不执行生产操作。
4. 本文档与具体 Prompt、Skill、门禁冲突时，以更严格的安全边界为准，并修正冲突资产。
5. 示例、报告和审计材料不得包含真实客户敏感信息、生产数据、密钥或生产配置。
