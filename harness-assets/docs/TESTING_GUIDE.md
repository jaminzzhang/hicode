# TESTING_GUIDE

## 1. 文档定位

本文档记录目标项目的测试规则、测试数据要求、Mock 策略和核心逻辑验证口径。

- 维护对象：测试负责人、研发负责人
- 更新时机：测试策略变化、缺陷复盘、核心场景变化后
- 主要使用者：TDD、辅助编码、提交检查、核心场景测试 Agent
- 当前版本：V1 初版规则，具体测试框架和覆盖率阈值需在试点仓库中补充

## 2. 基本原则

| 规则 | 说明 | V1 处理方式 |
|---|---|---|
| 测试前置 | 编码前先明确测试目标、场景、边界、异常、Mock 和断言 | TDD 必做项 |
| 核心逻辑必须有单元测试 | 金额、日期、状态、责任、权限、外部调用等核心逻辑必须覆盖 | 代码审查重点 |
| 测试必须覆盖正常、边界、异常 | 不只验证 happy path | TDD 和提交检查关注 |
| 外部系统必须 Mock | 单元测试不访问真实外部系统 | 核心规则 |
| 单元测试不得访问真实数据库 | 需求文档明确要求 | 核心规则 |
| 测试数据不得包含真实敏感信息 | 客户、保单、账号、Token 等必须脱敏或使用虚构数据 | 安全阻断 |

## 3. 测试分层

| 测试类型 | 适用范围 | 必要条件 | 执行方式 |
|---|---|---|---|
| 单元测试 | 业务规则、金额、日期、状态、权限、异常处理 | 必须可重复、可隔离、无真实外部依赖 | 本地或 CI 执行，外部依赖 Mock |
| 集成测试 | 模块协作、接口协议、数据访问适配 | 使用受控测试环境和脱敏数据 | 按试点项目确认 |
| 回归测试 | 核心业务场景和历史缺陷防范 | 覆盖核心场景库和缺陷案例 | 按版本或需求执行 |
| 发布验证 | 发布后关键路径验证 | 有明确验证点和回滚观察项 | 发布负责人确认 |

## 4. TDD 输出要求

TDD 设计至少包含：

1. 测试目标。
2. 测试范围。
3. 测试场景清单。
4. Given-When-Then 用例。
5. 边界场景。
6. 异常场景。
7. Mock 策略。
8. 测试数据。
9. 断言规则。
10. 单元测试代码或测试代码建议。
11. 上下文更新建议。

## 5. 必测场景类型

| 场景类型 | 覆盖要求 | 示例关注点 |
|---|---|---|
| 正常场景 | 覆盖主要业务路径和预期输出 | 投保成功、批改成功、退保成功、收付费成功 |
| 边界场景 | 覆盖金额、日期、状态、数量、权限边界 | 起保日、终保日、越界、闰年、短期费率边界 |
| 异常场景 | 覆盖输入非法、外部失败、状态非法、权限不足 | 外部系统超时、重复请求、禁止状态流转 |
| 历史缺陷场景 | 覆盖已复盘的缺陷模式 | 回归历史缺陷触发条件 |
| 安全场景 | 覆盖敏感信息、权限和日志安全 | 脱敏、越权、日志泄露 |

## 6. 核心业务测试关注点

| 业务主题 | 测试关注点 | 风险等级 |
|---|---|---|
| 金额 | 金额计算、舍入、精度、退费、账务一致性 | 高 |
| 日期 | 起保日、终保日、保险期间、越界、闰年 | 高 |
| 状态流转 | 允许流转、禁止流转、异常回退、已理赔状态 | 高 |
| 责任判断 | 产品责任、保险责任、核保和理赔相关判断 | 高 |
| 权限 | 查询、批改、配置、客户信息访问 | 高 |
| 外部调用 | 超时、失败、重试、幂等、补偿 | 高 |
| 客户隐私 | 脱敏、日志、示例数据、权限访问 | 高 |

## 7. Mock 策略

| 依赖类型 | Mock 要求 | 禁止事项 |
|---|---|---|
| 外部系统接口 | 模拟成功、失败、超时、异常响应 | 单元测试访问真实外部系统 |
| 数据库 | 使用内存、桩对象或受控测试替身，按项目技术栈确认 | 单元测试访问真实数据库 |
| 时间 | 固定系统时间或注入时间源 | 测试依赖当前真实时间导致不稳定 |
| 随机数/序列号 | 固定或可预测输入 | 测试结果不可重复 |
| 权限/登录态 | 构造受控身份和角色 | 使用真实账号、Cookie、Session |
| 文件/配置 | 使用测试配置或临时资源 | 读取生产配置或密钥文件 |

## 8. 断言规则

| 数据类型 | 断言要求 | 注意事项 |
|---|---|---|
| 金额 | 使用精确数值断言；需求文档示例要求 BigDecimal | 不使用浮点近似替代业务金额断言 |
| 日期 | 明确断言起止边界、格式和特殊日期 | 覆盖起保日、终保日、越界和闰年 |
| 状态 | 断言当前状态和禁止流转结果 | 不能只断言方法返回成功 |
| 异常 | 断言异常类型、业务错误码或错误信息摘要 | 不依赖完整敏感错误文本 |
| 外部调用 | 断言调用次数、关键参数、异常处理路径 | 不输出敏感请求/响应 |
| 日志/审计 | 断言必要追踪信息存在且敏感信息不泄露 | 具体方式按项目框架确认 |

## 9. 测试数据规则

| 数据类型 | 构造要求 | 脱敏要求 |
|---|---|---|
| 客户信息 | 使用虚构姓名、证件、手机号、邮箱、地址 | 必须脱敏或虚构 |
| 保单信息 | 使用虚构保单号、产品、期间、状态 | 不使用真实保单号 |
| 金额 | 覆盖 0、最小值、边界值、大额、舍入值 | 不使用真实账务数据 |
| 日期 | 覆盖起保日、终保日、跨月、跨年、闰年、越界 | 不使用真实客户日期 |
| 外部系统响应 | 构造成功、失败、超时、空响应、重复响应 | 不使用真实接口返回 |
| 权限身份 | 构造测试角色和权限 | 不使用真实账号、Token、Session |

12. 收付费交互。
13. 渠道交互。
14. 客户信息交互。
15. 报表/监管数据输出。

## 11. 覆盖率与提交要求

| 指标 | V1 要求 | 说明 |
|---|---|---|
| 单元测试结果 | 提交前应通过 | 失败需说明原因和修复计划 |
| 核心逻辑覆盖 | 必须覆盖 | 金额、日期、状态、责任、权限、外部调用 |
| 正常/边界/异常覆盖 | 必须覆盖 | TDD 报告和代码审查检查 |
| 外部依赖 Mock | 必须覆盖 | 不访问真实外部系统 |
| 敏感信息测试 | 必须检查 | 确认测试数据和日志不泄露敏感信息 |

## 12. 提交检查输入

提交检查或创建 MR 前，应准备：

1. 关联需求/故事链接。
2. 编码计划。
3. TDD 测试设计报告。
4. 单元测试结果。
5. 覆盖率结果，若项目已有统计方式。
6. 代码审查结果。
7. SQL / 配置 / 脚本变更清单。
8. 敏感信息检查结果。
