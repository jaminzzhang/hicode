# CODING_RULES

## 1. 文档定位

本文档记录目标项目的编码规则，供编码计划、辅助编码、代码审查和提交检查引用。

- 维护对象：技术负责人、架构师、模块 Owner
- 更新时机：编码规范变化、架构约束变化、质量问题复盘后
- 主要使用者：编码计划、辅助编码、代码审查、提交检查 Agent
- 当前版本：V1 初版规则，具体技术栈适配项需在试点仓库中补充

## 2. 基本原则

| 规则 | 说明 | V1 处理方式 |
|---|---|---|
| 不修改无关代码 | 变更范围必须受需求上下文、编码计划和测试目标约束 | 代码审查和提交检查提醒 |
| 不删除已有测试 | 除非有明确原因和负责人确认 | P0/P1 风险 |
| 不绕过异常处理 | 不允许吞异常、隐藏错误或删除必要校验 | P0/P1 风险 |
| 不直接访问生产配置 | Agent 不读取 `.env`、密钥文件或生产配置 | 安全阻断 |
| 不输出敏感信息 | 禁止输出客户敏感信息、Token、Cookie、Session、连接串、生产 IP 等 | 安全阻断 |
| 核心逻辑必须可测试 | 涉及金额、日期、状态、责任、权限、外部调用时必须有测试设计 | 代码审查提醒，核心场景建议阻断 |

## 3. 分层与职责

| 规则 | 适用范围 | 说明 | 风险等级 |
|---|---|---|---|
| Controller 不写业务逻辑 | 接口层 / Web 层 | Controller 只做参数接收、基础校验、响应组装和调用应用服务 | 高 |
| 业务规则集中在明确职责层 | Service / Domain / Application 层，按试点项目架构确认 | 金额、日期、状态、责任、核保、理赔等规则不得散落在工具类或接口层 | 高 |
| 外部系统适配独立封装 | 渠道、收付费、客户信息、监管/报表等外部调用 | 调用、超时、异常、日志、重试和幂等策略应集中管理 | 高 |
| 数据访问与业务逻辑分离 | Repository / DAO / Mapper 层，按试点项目架构确认 | 不在数据访问层承载复杂业务判断 | 中 |
| 公共工具不得隐藏业务语义 | 工具类、公共方法 | 金额、日期、责任等领域规则应有清晰命名和来源 | 中 |

## 4. 金额与精度

| 规则 | 适用范围 | 说明 | 风险等级 |
|---|---|---|---|
| 金额计算必须使用精确数值类型 | 保费、退保金额、收付费、账务处理 | 需求文档示例要求使用 BigDecimal；其他技术栈应使用等价精确数值方案 | 高 |
| 金额规则必须可追溯 | 保费、短期费率、退保金额、账务 | 公式、舍入、精度、币种和边界需在需求或领域文档中说明 | 高 |
| 禁止使用浮点数表达金额 | 金额计算和断言 | 避免精度误差 | 高 |
| 金额变更必须补测试 | 涉及金额的业务逻辑 | 覆盖正常、边界、异常和舍入场景 | 高 |

## 5. 日期与时间

| 规则 | 适用范围 | 说明 | 风险等级 |
|---|---|---|---|
| 日期计算必须明确起止边界 | 起保日、终保日、退保日、批改日、保险期间 | 必须说明是否含首含尾、跨日、时区和闰年处理方式 | 高 |
| 日期规则不得隐式默认 | 业务规则、查询条件、批处理 | 默认值、截断、格式转换必须明确来源 | 高 |
| 日期变更必须补边界测试 | 日期相关需求 | 覆盖起保日、终保日、越界和闰年场景 | 高 |

## 6. 状态流转

| 规则 | 适用范围 | 说明 | 风险等级 |
|---|---|---|---|
| 状态集合必须明确 | 保单状态、流程状态、缺陷状态 | 状态值、含义和归属对象应写入上下文 | 高 |
| 状态流转必须校验 | 投保、批改、退保、理赔前置 | 禁止流转和异常回退需明确处理 | 高 |
| 涉及已理赔保单必须人工确认 | 理赔前置、批改、退保 | 是否允许继续处理以及限制条件必须由负责人确认 | 高 |

## 7. 异常与日志

| 规则 | 适用范围 | 说明 | 风险等级 |
|---|---|---|---|
| 外部调用必须有异常处理 | 渠道、收付费、客户信息等外部系统 | 包含超时、失败、重试、降级或补偿策略 | 高 |
| 不允许吞异常 | 所有业务逻辑 | 捕获异常后必须有明确处理、日志或抛出策略 | 高 |
| 核心交易必须记录业务追踪标识 | 核心交易和跨系统流程 | 需求文档示例要求记录 `businessTraceId`；具体字段名可在试点项目确认 | 中 |
| 日志不得泄露敏感信息 | 所有日志 | 客户信息、保单号、客户号、Token、连接串等必须脱敏或禁止输出 | 高 |

## 8. 事务与幂等

| 规则 | 适用范围 | 说明 | 风险等级 |
|---|---|---|---|
| 涉及收付费和外部调用必须考虑幂等 | 收付费、退保、批改、渠道交互 | 幂等键、重复请求处理、补偿方式需明确 | 高 |
| 大事务需标注风险 | 批量导入、批改、收付费、状态流转 | 长事务、跨系统事务和循环调用需在编码计划中说明 | 中 |
| 失败补偿必须明确 | 外部调用、账务处理、批量处理 | 明确重试、回滚、人工处理或任务补偿策略 | 高 |

## 9. 外部系统调用

| 规则 | 适用范围 | 说明 | 风险等级 |
|---|---|---|---|
| 必须设置超时 | 所有外部系统调用 | 超时阈值按试点项目确认，不能无限等待 | 高 |
| 必须记录必要日志 | 外部调用请求和响应摘要 | 只记录可审计且脱敏后的信息 | 中 |
| 必须处理异常响应 | 失败、超时、错误码、空响应 | 明确业务结果、重试或人工处理路径 | 高 |
| 接口变更必须更新上下文 | 接口新增、字段变更、协议变更 | 更新 `docs/PROJ_CONTEXT.md` 和相关需求上下文 | 中 |

## 10. 权限与敏感信息

| 规则 | 适用范围 | 说明 | 风险等级 |
|---|---|---|---|
| 禁止读取密钥和生产配置 | `.env`、密钥文件、生产配置文件 | 除非明确确认安全、非生产上下文，否则 Agent 不读取 | 高 |
| 禁止输出认证凭证 | Token、Cookie、Session、内部密钥 | 任何报告、示例、日志均不得输出 | 高 |
| 客户敏感信息必须脱敏 | 姓名、身份证号、手机号、银行卡号、保单号、客户号、地址、邮箱 | 进入 Prompt、报告、示例前必须脱敏 | 高 |
| 权限逻辑必须可审查 | 查询、批改、配置、发布、客户信息访问 | 涉及权限时必须在需求和代码审查中标注 | 高 |

## 11. 配置、SQL 与脚本

| 规则 | 适用范围 | 说明 | 风险等级 |
|---|---|---|---|
| 配置变更必须说明发布影响 | 配置新增、修改、删除 | 提交检查和发布检查需记录影响范围 | 中 |
| SQL 变更必须说明回滚方案 | DDL、DML、初始化数据 | 发布前检查需关注回滚和数据影响 | 高 |
| 脚本变更必须说明执行条件 | 批处理、修复脚本、迁移脚本 | 不允许 Agent 直接操作生产环境 | 高 |

## 12. Agent 辅助编码约束

辅助编码 Agent 必须遵守：

1. 基于 `docs/PRD_CONTEXT.md`、`docs/PROJ_CONTEXT.md`、编码计划和 TDD 输出执行。
2. 不扩大改动范围。
3. 不删除已有测试。
4. 不绕过异常、日志、权限和审计。
5. 不输出敏感信息。
6. 涉及核心逻辑时同步补充或建议补充单元测试。
7. 发现新上下文时提出文档更新建议，不直接把未确认内容当作事实。

## 13. 禁止事项

| 禁止项 | 原因 | 例外条件 |
|---|---|---|
| 自动合并 MR / PR | 需要人工 Review 和责任确认 | 无 |
| 自动发布 | 发布决策必须人工确认 | 无 |
| 修改生产配置 | 生产变更必须走正式流程 | 无 |
| 直接操作生产环境 | 防止生产事故 | 无 |
| 输出生产密钥或客户敏感信息 | 安全红线 | 无 |
| 为通过测试而删除断言或测试 | 破坏质量闭环 | 无 |

## 14. 上下文更新要求

| 触发条件 | 建议更新文档 |
|---|---|
| 新增或确认业务规则 | `docs/DOMAIN_KNOWLEDGE.md`、`docs/PRD_CONTEXT.md` |
| 模块、接口、表、状态、外部依赖变化 | `docs/PROJ_CONTEXT.md` |
| 新增编码约束或例外 | `docs/CODING_RULES.md` |
| 发现历史缺陷模式 | `docs/DEFECT_CASES.md` |
| 发生重要架构决策 | `docs/ADR/` |
