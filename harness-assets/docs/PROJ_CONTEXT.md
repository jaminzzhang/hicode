# PROJ_CONTEXT

## 1. 文档定位

本文档描述目标项目的研发上下文，包括模块结构、关键流程、接口、数据、约束和历史风险。它不替代根目录 `CONTEXT.md` 类型的术语表，也不记录单个需求的临时上下文。

- 维护对象：研发负责人、架构师、模块 Owner
- 更新时机：模块职责变化、核心流程变化、关键接口变化、重要技术约束变化后
- 主要使用者：编码计划、TDD、辅助编码、代码审查 Agent
- 使用边界：单需求信息写入 `docs/PRD_CONTEXT.md`，领域知识写入 `docs/DOMAIN_KNOWLEDGE.md`

## 16. Agent 使用提示

Agent 读取本文档时应重点确认：

1. 本次需求涉及哪些模块、接口、表、配置和外部依赖。
2. 是否涉及金额、日期、状态流转、权限、客户隐私、事务、幂等或外部调用。
3. 是否存在历史风险点或相关 ADR。
4. 是否需要提出 `docs/PRD_CONTEXT.md`、`docs/DOMAIN_KNOWLEDGE.md`、`docs/DEFECT_CASES.md` 或 ADR 更新建议。
