# Insurance Domain Review Rules

## 1. 定位

本文档记录保险/金融核心系统业务规则相关的代码审查细则。

- 上级总则：`docs/REVIEW_RULES.md`
- 触发条件：diff 涉及投保、批改、退保、核保、理赔前置、收付费、责任、费率、保单/批单状态或客户权益
- 使用边界：只审查已确认或可追溯的业务规则，不编造未确认规则

## 2. 规则来源

业务判断必须能追溯到以下来源之一：

1. `docs/PRD_CONTEXT.md` 中已确认的单需求规则。
2. `docs/DOMAIN_KNOWLEDGE.md` 中已确认的领域规则。
3. 已复盘的 `docs/DEFECT_CASES.md`。
4. 需求、产品、业务、研发或测试负责人提供的明确输入。

缺少来源时，结论应为 `待确认` 或 P1/P2 风险；涉及客户权益、金额、保单状态或权限时可提升为 P0/P1。

## 3. 核心业务主题

| 主题 | 必查问题 | 常见等级 |
|---|---|---|
| 投保 | 产品、计划、责任、期间、保费、核保、客户身份和渠道规则是否完整 | P0/P1 |
| 批改 | 是否校验可批改状态、批改生效日、差额保费、批单状态和原保单关系 | P0/P1 |
| 退保 | 是否校验退保资格、退保日期、退费金额、已理赔或已出险限制 | P0/P1 |
| 核保 | 核保结论、人工审批、拒保/延期/加费规则是否有依据 | P0/P1 |
| 理赔前置 | 是否处理已报案、已理赔、责任免除或赔付状态对后续操作的影响 | P0/P1 |
| 收付费 | 是否防止重复扣款、重复退费、对账不一致或冲正遗漏 | P0 |
| 责任与权益 | 保险责任、等待期、免赔、限额、责任期间和权益影响是否明确 | P0/P1 |
| 渠道交互 | 渠道订单、回调、幂等、签名、重复通知和状态同步是否正确 | P1 |

## 4. 金额与费率

1. 金额必须使用精确数值类型，不得使用浮点类型处理保费、退费、赔付或对账金额。
2. 必须明确公式、精度、舍入、币种、含税/不含税和边界值。
3. 批改、退保、冲正、对账必须检查正负方向和重复处理。
4. 金额差异、尾差和补退费规则不清楚时，必须列为待确认。

## 5. 日期与期间

1. 必须明确保险期间、批改生效日、退保日、等待期、责任起止、宽限期等日期规则。
2. 必须检查含首含尾、跨月、闰年、时区、自然日/工作日等边界。
3. 日期边界影响客户权益或金额时，默认按 P0/P1 审查。

## 6. 状态流转

1. 必须校验允许流转和禁止流转。
2. 不得只依赖前端状态或页面入口控制。
3. 状态变更必须考虑并发、重复请求、回调乱序和异常回退。
4. 保单、批单、支付、清算、核保、理赔前置状态不一致时，必须给出 P0/P1 风险判断。

## 7. 测试关注点

1. 每个核心业务规则至少有正常、边界、异常用例或明确豁免。
2. 金额、日期、状态、权限、幂等和外部回调必须有断言。
3. 历史缺陷相似场景必须加入回归建议。
4. 测试数据必须脱敏，不得使用真实客户资料或生产数据。
