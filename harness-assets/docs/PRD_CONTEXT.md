# PRD_CONTEXT

## 1. 文档定位

本文档记录单个需求的上下文，帮助 Agent 在需求评审、编码计划、TDD、代码审查和发布检查时理解需求边界。

- 维护对象：需求负责人、研发负责人、测试负责人
- 更新时机：需求澄清、评审结论变化、范围变化、测试或发布关注点变化后
- 主要使用者：需求评审、编码计划、TDD、代码审查、发布前检查 Agent
- 使用边界：仅记录当前需求；可复用领域知识写入 `docs/DOMAIN_KNOWLEDGE.md`，仓库/模块上下文写入 `docs/PROJ_CONTEXT.md`
