# DEFECT_CASES

## 1. 文档定位

本文档记录历史缺陷案例、触发原因、防范规则和可复用检查点，用于需求评审、TDD、核心场景测试、代码审查和发布检查。

- 维护对象：测试负责人、研发负责人、模块 Owner
- 更新时机：缺陷复盘、线上问题复盘、测试遗漏复盘后
- 主要使用者：需求评审、TDD、代码审查、核心场景测试 Agent
- 当前版本：V1 初版模板，不记录真实客户敏感信息

## 2. 缺陷分级

| 等级 | 含义 | 处理要求 |
|---|---|---|
| P0 | 生产事故、数据错误、安全漏洞、重大资损或合规问题 | 必须复盘并形成防范规则 |
| P1 | 核心功能异常、边界错误、性能问题或高风险漏测 | 应复盘并补测试或 Review 规则 |
| P2 | 一般功能问题、可维护性问题、局部测试遗漏 | 建议补充防范规则 |
| P3 | 体验、文案、风格或低风险问题 | 可选沉淀 |

## 3. 缺陷分类

| 分类 | 说明 | 常见来源 | 建议沉淀位置 |
|---|---|---|---|
| 需求理解 | 需求目标、范围、验收标准理解不一致 | 需求评审遗漏 | `docs/PRD_CONTEXT.md` |
| 业务规则 | 金额、日期、状态、责任、核保、理赔规则错误 | 规则未确认或实现偏差 | `docs/DOMAIN_KNOWLEDGE.md` |
| 边界条件 | 起止日期、金额边界、数量边界、状态边界遗漏 | TDD 不完整 | `docs/TESTING_GUIDE.md` |
| 状态流转 | 允许/禁止流转、异常回退或已理赔状态处理错误 | 上下文缺失 | `docs/PROJ_CONTEXT.md` |
| 外部依赖 | 渠道、收付费、客户信息接口异常处理不足 | Mock 或集成测试不足 | `docs/PROJ_CONTEXT.md` |
| 数据质量 | 数据重复、缺失、不一致或口径错误 | 数据校验不足 | `docs/TESTING_GUIDE.md` |
| 安全隐私 | 敏感信息泄露、权限绕过、密钥提交 | 安全规则未执行 | `docs/CODING_RULES.md` |
| 发布遗漏 | SQL、配置、制品、需求、验证点或回滚遗漏 | 发布检查不足 | `docs/RELEASE_GUIDE.md` |

## 5. 防范规则清单

| 规则编号 | 缺陷类型 | 防范要求 | 关联检查点 | 适用工作流 |
|---|---|---|---|---|
| DC-RULE-001 | 金额 | 金额计算必须明确公式、精度、舍入和边界测试 | 金额 Review、TDD 边界测试 | 编码计划、TDD、代码审查 |
| DC-RULE-002 | 日期 | 日期需求必须覆盖起保日、终保日、越界和闰年 | 日期断言、边界测试 | TDD、核心场景测试 |
| DC-RULE-003 | 状态流转 | 状态变更必须校验允许流转和禁止流转 | 状态流转 Review | 编码计划、代码审查 |
| DC-RULE-004 | 外部依赖 | 外部调用必须覆盖成功、失败、超时和重复响应 | Mock 策略 | TDD、代码审查 |
| DC-RULE-005 | 敏感信息 | 测试数据、日志和报告不得包含真实敏感信息 | 敏感信息检查 | 代码审查、提交检查、发布检查 |

## 10. 复盘输出要求

缺陷复盘后至少输出：

1. 缺陷等级和影响范围。
2. 根因分析。
3. 漏检环节。
4. 修复动作。
5. 防范规则。
6. 应补测试。
7. 应更新的 Harness 文档。
8. 是否需要新增或更新 ADR。
