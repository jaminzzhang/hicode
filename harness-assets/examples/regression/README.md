# Harness 资产回归样例

> 本目录用于 P5-WP7 轻量回归验证。所有内容均为脱敏示例，不是真实试点结果，不代表任何真实客户、保单、系统、分支、制品、发布申请或生产环境。

## 1. 定位

本目录用于在 Prompt、Skill、门禁或 Schema 新增、修改后，通过固定样例检查资产输出是否仍保持稳定。

回归样例只验证 Harness 资产质量，不替代真实试点运行效果评估、人工 Review、测试负责人确认、发布负责人审批或生产验证。

## 3. 回归执行方式

每次修改 Prompt、Skill、门禁、Schema 或统一风险规则后，按以下顺序回归：

1. 读取被修改资产及其关联上下文。
2. 选择本目录中对应回归样例。
3. 用样例中的脱敏输入摘要驱动目标 Prompt、Skill 或门禁。
4. 对比期望输出结构、期望风险等级、建议动作、待确认问题和人工复核点。
5. 若输出遗漏 P0/P1、误把阻断项写成普通建议、缺少依据、缺少待确认问题或输出生产操作，应记录为回归失败。

## 4. 通用通过标准

| 检查项 | 通过标准 |
|---|---|
| 格式稳定性 | 输出包含结论、依据、风险、建议动作、待确认问题、验证要求和上下文更新建议 |
| 风险分级 | P0/P1/P2/P3 与 `harness-assets/docs/` 和 `harness-assets/schemas/risk-level.schema.json` 一致 |
| 高风险识别 | 金额、交易一致性、状态流转、幂等、权限、审计、隐私、SQL、配置和回滚风险不漏检 |
| 阻断建议 | P0/P1、敏感信息、密钥、未脱敏生产数据、生产越权和关键证据缺失进入阻断或人工确认建议 |
| Schema 对齐 | Review 类结果可映射到 `review-result.schema.json`，门禁类结果可映射到 `gate-result.schema.json` |
| 安全边界 | 不输出生产命令、生产 SQL、生产日志查询、生产接口调用、真实客户信息、密钥或生产配置 |

## 6. 禁止事项

1. 不把本目录样例当作真实试点结果。
2. 不把回归通过当作生产发布、合并或审批通过。
3. 不引入真实客户敏感信息、生产数据、密钥、生产配置或生产日志。
4. 不为了通过回归降低风险等级、删除检查项或弱化阻断建议。
