# 提交检查 Skill 示例案例

> 本示例为 P4-WP7 脱敏结构示例，不是真实试点结果，不代表任何真实客户、保单、系统、分支或发布记录。

## 1. 适用 Skill

`harness-assets/skills/pre-commit-check/SKILL.md`

## 2. 脱敏输入摘要

提交目标：批改单生效日期规则调整进入 MR 前检查。

示例输入：

1. 当前分支：`feature/demo-endorsement-date-rule`
2. 目标分支：`develop-demo`
3. 关联需求：`REQ-DEMO-2026-001`
4. 代码审查报告：示例中 P1 已修复，剩余 P2 审计字段命名建议。
5. 测试结果摘要：示例假设单元测试通过，集成测试由 CI 执行。
6. SQL/配置：不涉及数据库结构变更，不涉及生产配置。

## 3. 处理重点

1. 提交检查默认消费代码审查报告。
2. 检查分支、需求关联、测试、敏感信息、SQL/配置和 MR 描述。
3. 不自动提交、不推送、不合并。
4. 示例命令只展示结构，不在本仓库执行目标项目命令。

### 4.7 MR / PR 描述草稿

#### 变更摘要

调整指定批改原因的生效日期校验规则，允许在保单起保日至申请日期之间选择回溯日期。

#### 测试与验证

1. 日期边界单元测试。
2. 终态保单限制测试。
3. 审计事件字段测试。

#### 风险与回滚关注点

1. 需关注批改单状态流转和审计字段。
2. 回滚方式为恢复原日期校验规则并保留已生成审计记录。

#### 待确认事项

1. CI 链接。
2. 审计字段命名确认记录。

## 5. 人工复核关注点

1. 是否存在未关闭 P0/P1 Review 问题。
2. CI、单测、敏感信息扫描是否真实执行。
3. MR 描述是否准确列明风险和回滚关注点。

## 6. 禁止误用说明

1. 不得把本示例当作真实提交许可。
2. 不得自动执行 `git commit`、`git push` 或合并操作。
3. 不得读取 `.env`、密钥文件或生产配置补充检查。
