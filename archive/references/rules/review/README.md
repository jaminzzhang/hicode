# Review Rules

## 定位

本目录保存代码审查、提交检查、专项审查和合并前建议规则。它服务 `hicode:review`，不替代人工 Review、MR/PR 审批或合并决策。

## 必要输入

1. 固定基准点、diff、分支、Commit、MR/PR 或工作区变更范围。
2. 需求来源、编码计划、TDD 报告、测试结果、CI 或扫描结果。
3. 目标项目编码、测试、Review、缺陷和发布上下文。
4. SQL、配置、脚本、权限、客户隐私或外部接口变更清单。

## 审查轴

| 审查轴 | 规则 |
|---|---|
| 需求轴 | 变更是否符合需求目标、范围、业务规则、验收标准和负责人确认 |
| 规范轴 | 变更是否符合编码、测试、安全、Review、ADR 和项目约定 |
| 证据轴 | 测试、CI、扫描、人工确认和上下文证据是否足以支撑结论 |

## 专项触发

| 触发条件 | 检查重点 |
|---|---|
| Java、Spring、事务、异常、日志、MyBatis/JPA、消息、批处理 | 分层、事务、异常、幂等、性能、SQL 和日志脱敏 |
| SQL、DDL、DML、索引、数据修复、配置脚本 | 影响范围、回滚、幂等、执行顺序、数据量和审批证据 |
| 权限、认证、鉴权、密钥、日志、客户信息、外部接口 | 越权、敏感信息、脱敏、审计、监管和生产越权 |
| 投保、批改、退保、核保、理赔前置、收付费、责任、费率、状态 | 保险核心业务规则、金额、状态流转、幂等和历史缺陷 |

## 阻断建议

以下情况必须进入阻断建议项：

1. 安全红线、密钥、未脱敏客户信息或未脱敏生产数据。
2. 金额、保单、客户权益、收付费、责任判断、状态流转可能错误。
3. P0/P1 问题未关闭或无法判断。
4. 测试失败、核心场景漏测或为了通过检查删除测试/降低断言。
5. 自动合并、自动发布、生产配置修改或生产操作诉求。

## 输出要求

输出应使用 `references/templates/review/review-report.md`，并包含：

1. 建议结论和最高风险。
2. 审查范围和基准点。
3. 依据与未读取材料。
4. 双轴审查结果。
5. 问题清单、阻断建议项和普通风险提示项。
6. 测试补充建议。
7. 受限命令执行记录。
8. 待确认问题和上下文更新建议。
