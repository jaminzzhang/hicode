# Security Review Rules

## 1. 定位

本文档记录权限、认证、脱敏、日志、密钥、外部接口和客户敏感信息相关的代码审查细则。

- 上级总则：`docs/REVIEW_RULES.md`
- 触发条件：diff 涉及权限、登录态、客户信息、日志、配置、外部接口、安全校验或敏感信息
- 使用边界：只提供审查规则和风险建议，不替代安全负责人最终判断

## 2. 敏感信息

不得读取、输出、写入示例或提交：

```text
客户姓名
身份证号
手机号
银行卡号
保单号
客户号
地址
邮箱
Token
Cookie
Session
数据库连接串
生产 IP
内部密钥
```

发现未脱敏敏感信息时，建议等级为 P0，并要求先脱敏再继续处理。

## 3. 权限与认证

| 检查项 | 未满足时建议等级 |
|---|---|
| 缺少登录态、角色、机构、渠道或数据权限校验 | P0/P1 |
| 仅前端控制权限，后端无校验 | P0 |
| 批改、退保、支付、配置、客户信息查询绕过审批或授权 | P0 |
| 越权访问他人保单、客户资料或业务单据 | P0 |
| 权限异常无审计记录 | P1 |

## 4. 日志、报文与脱敏

1. 日志、异常、Review 报告、测试数据不得包含敏感信息原文。
2. 外部接口请求和响应日志必须按字段脱敏。
3. 不得为了排查问题临时输出密钥、连接串、Cookie、Session 或完整客户资料。
4. 错误信息不得暴露内部路径、SQL、表名、密钥位置或生产连接信息。

## 5. 注入与输入校验

1. SQL 拼接、动态排序、动态表名或动态条件必须检查注入风险。
2. 外部输入必须校验类型、长度、枚举、范围和必填项。
3. 文件上传、批量导入和报文解析必须检查格式、大小、编码和异常路径。
4. 签名、回调、渠道请求必须校验来源、时间戳、nonce 或幂等键。

## 6. 配置与密钥

1. 不得读取 `.env`、密钥文件、生产配置文件或生产凭证。
2. 不得把密钥、Token、连接串写入代码、测试、日志、Prompt、示例或报告。
3. 配置变更必须说明环境、默认值、回滚方式和发布影响。
4. 生产配置修改和发布由既有流程审批，Agent 不得自动执行。

## 7. 外部服务与数据出境

1. 向外部模型、外部 API 或第三方服务提交信息前，必须确认数据已脱敏。
2. 不得提交未脱敏生产日志、客户数据、密钥或生产配置。
3. 监管报送、客户隐私和数据最小化要求不清楚时，列为待确认。
