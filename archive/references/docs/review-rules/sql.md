# SQL Review Rules

## 1. 定位

本文档记录 SQL、DDL、DML、索引、批量更新、数据修复和配置脚本相关的代码审查细则。

- 上级总则：`docs/REVIEW_RULES.md`
- 触发条件：diff 涉及 SQL、数据库脚本、表结构、索引、数据修复、配置变更或批处理
- 使用边界：只审查 SQL 和数据变更风险，不替代发布审批或 DBA 审核

## 2. DDL 检查

| 检查项 | 风险 | 常见等级 |
|---|---|---|
| 新增、删除、修改字段未说明兼容性 | 应用启动失败或历史数据错误 | P0/P1 |
| 字段类型、长度、精度不匹配业务含义 | 金额、日期、状态或标识错误 | P0/P1 |
| 非空字段缺少默认值或回填方案 | 发布失败或存量数据不可用 | P1 |
| 删除表、字段、索引无影响分析 | 不可逆变更 | P0/P1 |
| 表结构变化未同步上下文 | 后续 Agent 和人工 Review 失准 | P2 |

## 3. DML 与数据修复

1. 必须有明确 where 条件，不得出现无条件更新或删除。
2. 必须说明影响范围、预估行数、回滚方案和验证 SQL。
3. 涉及保单、批单、收付费、客户权益、状态字段的数据修复，默认按 P0/P1 审查。
4. 不得在脚本中包含未脱敏客户敏感信息。
5. 批量脚本必须考虑锁表、执行时间、分批策略和失败续跑。

## 4. 查询与索引

| 检查项 | 未满足时建议等级 |
|---|---|
| 新增高频查询无索引或索引不可用 | P1/P2 |
| 循环查库、N+1 查询或分页错误 | P1/P2 |
| 模糊查询、大范围扫描影响核心交易 | P1 |
| 使用函数导致索引失效 | P2 |
| 排序、分页缺少稳定字段 | P2 |

## 5. 事务与锁

1. 大批量更新不得放入不可控长事务。
2. 脚本必须说明执行窗口、锁风险和失败处理。
3. 涉及并发写入的表，必须检查唯一键、版本号、状态校验或幂等键。
4. 不得通过降低隔离要求掩盖一致性问题。

## 6. 发布与回滚

1. SQL 必须进入发布检查清单。
2. DDL、DML、配置脚本必须区分执行顺序和依赖关系。
3. 必须有回滚脚本或说明为什么不可回滚。
4. 发布后必须有验证 SQL 或业务验证点。
5. 生产执行必须走既有审批流程，Agent 不得自动执行生产 SQL。
