## 2. 脱敏输入摘要

需求编号：`REQ-DEMO-2026-001`

需求名称：批改单生效日期规则调整

已确认前置：

1. 仅 `地址变更`、`联系方式变更` 两类批改原因允许回溯。
2. 生效日期不得早于保单起保日，不得晚于申请日期。
3. 已退保、已终止、已满期保单不允许回溯批改。
4. 回溯批改不触发保费重算。
5. 必须记录回溯原因、操作人、操作时间和原始申请日期。

示例技术上下文：

1. 示例模块：`endorsement-service`
2. 示例规则类：`EndorsementEffectiveDateRule`
3. 示例接口：`POST /demo/endorsements`
4. 示例测试框架：JUnit 5

### 4.4 建议实现步骤

1. 增加规则类行为测试：允许指定批改原因选择保单起保日至申请日期之间的日期。
2. 增加边界测试：早于起保日、晚于申请日期、等于起保日、等于申请日期。
3. 增加状态限制测试：已退保、已终止、已满期保单不允许回溯。
4. 增加审计字段测试：回溯原因、操作人、操作时间和原始申请日期必须进入审计事件。
5. 实现规则类最小逻辑。
6. 接入接口层校验和错误码映射。
7. 在 GREEN 后整理命名和重复判断逻辑。
