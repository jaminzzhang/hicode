# 需求准入门禁

## 1. 定位

本门禁用于需求或故事进入开发前的提醒型检查，判断需求材料、需求评审结果、P0 澄清点、核心业务规则、验收标准和单需求上下文是否足以支撑后续编码计划。

本门禁不替代业务负责人、产品负责人、研发负责人或需求评审人的最终判断，不自动创建需求、不自动审批、不自动流转流程。

## 2. 适用场景

使用本门禁的场景：

1. 需求或故事准备进入开发前。
2. 需求评审 Skill 已输出报告，需要形成节点门禁检查结论。
3. 需求范围、核心业务规则、验收标准或 `docs/PRD_CONTEXT.md` 是否齐备需要统一检查。
4. 存在 P0/P1 澄清问题，需要判断是否阻断进入开发。

不适用场景：

1. 需求仍在草稿生成阶段，未进入评审。
2. 当前目标是生成需求评审报告；应先使用 `requirement-review` Skill。
3. 当前目标是判断是否可以开始编码；应使用 `coding-entry-gate.md`。
4. 输入包含未脱敏客户敏感信息、生产数据、密钥、`.env`、生产配置或生产凭证。

## 3. 输入材料

至少需要：

1. 需求或故事链接、编号、摘要或脱敏需求材料。
2. 需求评审 Skill 报告。
3. P0/P1/P2/P3 澄清问题清单及状态。
4. `docs/PRD_CONTEXT.md`，或已确认的等价单需求上下文材料。
5. 核心业务规则、验收标准和建议确认人记录。

缺少输入时，必须在报告中标注缺口，不得编造需求、规则、验收标准或负责人确认。

## 4. 建议结论规则

| 结论 | 使用条件 |
|---|---|
| 建议通过 | 检查对象明确；需求评审结论为通过或有条件通过；无未关闭 P0 澄清问题；核心业务规则和验收标准明确；`docs/PRD_CONTEXT.md` 或等价材料齐备；未发现敏感信息或生产越权 |
| 有条件通过 | 无阻断建议项，但存在 P1/P2 待确认、等价材料需补沉淀、验收标准需增强或负责人确认需归档 |
| 建议阻断 | 未关闭 P0；核心业务规则缺失；验收标准缺失；需求评审结论不建议进入开发；发现敏感信息、密钥、未脱敏生产数据或生产越权 |
| 待确认 | 缺少需求评审报告、检查对象不明确、输入材料无法确认、需求范围不清且无法判断风险等级 |

## 5. 检查项

| 检查项 | 通过条件 | 阻断或降级规则 |
|---|---|---|
| 检查对象明确 | 有需求或故事编号、摘要或链接 | 缺失时结论为待确认 |
| 需求评审报告 | 已提供且结论为通过或有条件通过 | 缺失时不得建议通过；结论不建议进入开发时建议阻断 |
| P0 澄清问题 | 全部关闭 | 任一未关闭 P0 建议阻断 |
| 核心业务规则 | 投保、批改、退保、核保、理赔前置、收付费、责任判断、状态流转等相关规则已明确或不适用 | 涉及相关域但规则缺失时建议阻断或待确认 |
| 验收标准 | 主流程、边界、异常和关键风险验收标准明确 | 缺失时建议阻断；部分不足时有条件通过 |
| 单需求上下文 | `docs/PRD_CONTEXT.md` 已生成，或等价材料已确认且明确后续补沉淀 | 缺失且无等价材料时待确认 |
| 敏感信息与生产越权 | 无未脱敏客户敏感信息、生产数据、密钥或生产操作诉求 | 命中时建议阻断并要求脱敏或转人工安全流程 |

## 6. 受限命令边界

本门禁默认不执行命令。只读取用户提供的脱敏材料、需求评审报告和上下文文档。

如后续工具化需要执行命令，必须先在权限审计矩阵中定义命令范围、执行条件和审计证据。本门禁不得执行生产连接、数据库变更、发布、回滚、自动提交、自动合并或删除文件。

## 7. 报告格式

输出报告必须基于 `_gate-template.md`，并保留以下节点专属检查项：

| 检查项 | 结果 | 依据 | 风险等级 | 建议动作 |
|---|---|---|---|---|
| 需求或故事输入明确 | 通过 / 不通过 / 待确认 |  |  |  |
| 需求评审报告结论可追溯 | 通过 / 不通过 / 待确认 |  |  |  |
| P0 澄清问题关闭 | 通过 / 不通过 / 待确认 |  |  |  |
| 核心业务规则明确 | 通过 / 不通过 / 待确认 / 不适用 |  |  |  |
| 验收标准明确 | 通过 / 不通过 / 待确认 |  |  |  |
| `docs/PRD_CONTEXT.md` 或等价材料齐备 | 通过 / 不通过 / 待确认 |  |  |  |
| 敏感信息和生产越权检查 | 通过 / 不通过 / 待确认 |  |  |  |

## 8. 安全约束

1. 不读取或输出真实客户姓名、证件号、手机号、银行卡号、保单号、客户号、地址、邮箱等敏感信息。
2. 不读取 `.env`、密钥文件、生产配置文件或生产凭证。
3. 不将需求准入结论写成最终审批。
4. 不因赶进度隐藏 P0/P1 风险或跳过负责人确认。

## 9. 验收要求

本门禁资产验收时应确认：

1. 覆盖需求评审、P0 澄清点、核心业务规则、验收标准和 `docs/PRD_CONTEXT.md`。
2. 使用统一门禁建议结论。
3. 阻断建议项与普通风险提示项分离。
4. 默认不执行命令。
5. 保留审计证据、待确认问题和下一步建议。
