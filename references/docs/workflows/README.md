# Workflows README

## 1. 目录定位

本目录保存目标项目的研发流程说明，用于解释每类 Harness 工作流的触发条件、输入、输出、下一步和关联 Prompt / Skill。

- 维护对象：AI 工程化小组、研发负责人、测试负责人
- 更新时机：工作流入口、输入输出、门禁规则或责任边界变化后
- 主要使用者：开发 Agent、试点研发人员、测试人员、技术负责人

## 3. 单个工作流文档结构

每个工作流文档建议包含：

1. 触发条件。
2. 必读上下文。
3. 输入材料。
4. 执行步骤。
5. 输出格式。
6. 质量标准。
7. 安全约束。
8. 下一步动作。

## 4. 通用安全约束

1. Agent 不读取生产账号、密钥、Token、Cookie、Session、数据库连接串、生产 IP 或内部密钥。
2. Agent 不向外部模型提交客户敏感信息。
3. Agent 不自动合并、不自动发布、不修改生产配置。
4. 涉及架构决策时，Agent 只能提出 ADR 判断或草稿，不能自动确认。
5. 所有输出应包含结论、依据、风险等级、建议动作、待确认问题和上下文更新建议。
6. Agent 只能提出上下文、规则、缺陷或 ADR 更新建议；正式沉淀必须由对应负责人确认。

## 5. 通用金融核心系统检查基线

所有工作流默认服务保险/金融核心系统研发。除非场景明确不适用，输出必须覆盖保险核心业务逻辑严谨性、金额精度、交易一致性、状态流转、幂等、权限、审计、客户隐私、监管合规、生产变更、回滚和发布准入；缺少输入时列入 `待确认`，不得编造保险业务规则。

## 6. Review 规则分层

代码审查相关工作流先读取 `docs/REVIEW_RULES.md` 总则，再根据 diff 按需读取 `docs/review-rules/` 下的 Java、SQL、安全或保险业务细则。不得把全部 Review 细则复制进 Prompt 或工作流正文。
