# DOMAIN_KNOWLEDGE

## 1. 文档定位

本文档沉淀目标项目可复用的意健险领域知识、业务术语、核心业务域和高风险主题。

- 维护对象：业务负责人、产品/BA、研发负责人、测试负责人
- 更新时机：业务规则确认、需求评审、缺陷复盘、核心流程变化后
- 主要使用者：需求评审、编码计划、TDD、核心场景测试、代码审查 Agent
- 当前版本：初版领域索引，具体业务规则多数仍为 `待确认`

## 2. 适用边界

当前内容是目标项目领域知识模板和初始索引。复制到试点仓库后，应由业务负责人、产品/BA、研发负责人和测试负责人基于已确认材料维护。

本文档只记录可复用领域知识和高风险主题，不记录单个需求的临时结论；单需求上下文应写入 `docs/PRD_CONTEXT.md`。

默认服务对象是保险/金融核心系统研发。本文档中的领域知识、风险主题和问题清单应按金融核心系统风险标准维护，重点覆盖保险核心业务逻辑严谨性、金额精度、交易一致性、状态流转、幂等、权限、审计、客户隐私、监管合规、生产变更、回滚和发布准入。

未确认的保险核心业务规则必须保留为 `待确认`，不得由 Agent 补编。

## 5. 高风险业务判断

以下事项属于金融核心系统高风险检查对象。Agent 只能辅助分析、生成建议或列出问题，不能替代负责人决策。

| 高风险事项 | 风险说明 | Agent 处理要求 | 状态 |
|---|---|---|---|
| 保险核心业务逻辑严谨性 | 影响投保、批改、退保、核保、理赔前置、收付费、责任判断等核心结果 | 标注为高风险；要求人工确认业务规则、例外场景和测试证据 | 待确认 |
| 保费计算 | 直接影响收费金额和业务结果 | 标注为高风险；要求人工确认规则、公式、边界和测试 | 待确认 |
| 退保金额计算 | 直接影响退费金额和账务处理 | 标注为高风险；要求人工确认计算口径、日期边界和账务影响 | 待确认 |
| 责任判断 | 影响承保、理赔或产品责任适用 | 标注为高风险；要求人工确认责任规则和例外场景 | 待确认 |
| 核保规则 | 影响是否承保或是否触发人工核保 | 标注为高风险；要求人工确认规则来源和触发条件 | 待确认 |
| 理赔规则 | 影响理赔相关判断和后续处理 | 标注为高风险；要求人工确认理赔边界和状态影响 | 待确认 |
| 账务处理 | 影响收付费、退费和财务一致性 | 标注为高风险；要求人工确认账务规则和幂等要求 | 待确认 |
| 客户隐私处理 | 涉及客户敏感信息 | 必须脱敏；禁止向外部模型提交敏感数据 | 已确认安全红线 |
| 权限控制 | 影响数据访问和操作边界 | 标注为高风险；要求人工确认权限模型和越权防护 | 待确认 |

## 6. 工程高风险主题

| 主题 | 适用场景 | 已确认要求 | 需继续确认 |
|---|---|---|---|
| 金额 | 保费、退保金额、收付费、账务处理 | 金额计算和断言需使用精确数值口径；Java 项目可使用 BigDecimal，其他技术栈使用等价方案 | 各业务域计算公式、舍入规则、币种和精度 |
| 日期 | 保险期间、起保日、终保日、退保日期、批改日期 | 日期计算必须明确起止边界；测试覆盖起保日、终保日、越界和闰年 | 各业务域日期口径、时区、含首含尾规则 |
| 状态流转 | 保单状态、流程状态、缺陷状态 | 涉及状态流转的需求必须标注风险 | 状态集合、允许流转、禁止流转、补偿逻辑 |
| 责任判断 | 产品责任、保险责任、理赔责任 | 属于人工主导高风险事项 | 责任规则来源、例外条件、历史缺陷 |
| 权限 | 查询、批改、发布、配置、客户信息访问 | 权限控制属于人工主导高风险事项 | 权限模型、角色清单、越权校验 |
| 客户隐私 | 客户资料、保单、客户号、身份信息 | 敏感信息必须脱敏；禁止泄露 | 脱敏规则、最小可用样例、日志落盘规则 |
| 外部系统调用 | 渠道、收付费、客户信息、监管或报表相关交互 | 外部系统调用必须考虑超时、异常处理和日志 | 接口清单、超时阈值、重试、幂等和降级 |
| 事务与幂等 | 收付费、退保、批改、外部交互 | 涉及事务和幂等的需求必须标注风险 | 幂等键、补偿规则、重复请求处理 |
