# PILOT_PROJECTS

## 1. 文档定位

本文档用于 P6-WP1，作为 V1 试点项目候选评估和选择建议模板。它帮助负责人从候选项目中选择 1-2 个适合试点 hicode V1 的项目，并为后续基线指标、使用记录和复盘报告提供脱敏追溯入口。

本文档不是试点执行台账，不记录真实人员名单、生产系统编号、生产地址、客户信息、保单号、密钥、生产配置或真实试点效果结果。

- 维护对象：AI 工程化小组、研发负责人、测试负责人、安全负责人
- 主要使用者：试点项目负责人、研发代表、测试代表、产品/BA 代表、架构师或技术负责人、发布负责人
- 关联资产：`references/docs/V1_ACCEPTANCE_CHECKLIST.md`、`references/docs/pilot/BASELINE_METRICS_PLAN.md`、`references/docs/pilot/PILOT_USAGE_LOG.md`

## 2. 使用规则

1. 每个候选项目使用一个脱敏 `pilot_project_id`，格式建议为 `PILOT-YYYY-NNN`，例如 `PILOT-2026-001`。
2. 项目名称可使用脱敏简称，不得填写真实内部敏感项目编号、客户名称、保单号、生产系统主键或不可脱敏链接。
3. 团队信息只记录角色，不记录真实姓名、工号、手机号、邮箱或 IM 账号。
4. 风险等级使用 P0 / P1 / P2 / P3；涉及客户隐私、生产越权、金额错误、核心状态流转错误、未经 Review 的 AI 代码合入等风险时不得降级。
5. 本模板只形成选择建议，不能替代项目负责人、研发负责人、测试负责人、安全负责人或发布负责人的最终确认。
6. 若候选项目缺少基线数据、需求边界、试点角色或安全脱敏条件，应标注为 `待确认` 或 `不建议纳入`，不得用推断补齐。

## 3. 候选项目总表

| pilot_project_id | 脱敏项目简称 | 项目类型 | 金融核心风险等级 | 推荐结论 | 主要适用场景 | 排除或限制原因 | 负责人确认状态 |
|---|---|---|---|---|---|---|---|
| PILOT-YYYY-001 | 待填写 | 需求 / 开发 / 测试 / 发布 / 综合 | P0 / P1 / P2 / P3 | 建议纳入 / 有条件纳入 / 不建议纳入 / 待确认 | 待填写 | 待填写 | 待确认 |

推荐结论口径：

| 结论 | 使用条件 |
|---|---|
| 建议纳入 | 场景匹配、材料可脱敏、角色齐备、基线口径可采集，且无未关闭 P0/P1 阻断风险 |
| 有条件纳入 | 基本匹配，但存在材料、角色、基线数据或安全审查缺口，需先完成补证据动作 |
| 不建议纳入 | 生产敏感信息不可脱敏、涉及未授权生产操作、核心风险不可控、负责人无法确认或试点范围不清 |
| 待确认 | 现有材料不足以判断，需补充候选项说明、风险评估或负责人确认 |

## 8. 禁止误用

1. 不记录真实客户敏感信息、生产数据、生产地址、生产账号、密钥、Token、Cookie、Session、连接串或保单号。
2. 不把候选项目建议写成真实试点已执行。
3. 不把 P5 仓库资产验收结论写成试点运行效果已达成。
4. 不替代项目负责人、研发负责人、测试负责人、安全负责人、发布负责人或业务负责人的最终确认。
5. 不为了纳入试点降低 P0/P1 风险等级、删除排除原因或隐藏证据缺口。
6. 不要求 Agent 自动合并、自动发布、连接生产、查询生产日志、执行生产 SQL 或修改生产配置。
