# Permissions

## 1. 权限等级

| 等级 | 名称 | 使用边界 |
|---|---|---|
| L0 | 只读分析 | 读取脱敏材料、上下文、规范、diff、报告或检查结果，输出分析结论 |
| L1 | 生成建议 | 输出计划、报告、测试设计、补丁草案、上下文更新建议、MR/PR 描述草稿或门禁建议 |
| L2 | 本地修改 | 在确认范围内修改本地代码、测试或必要文档；必须受当前 Skill、用户指令和工作包边界约束 |
| L3 | 受限命令 | 执行本地非生产测试、构建、静态检查、格式检查、覆盖率、敏感信息扫描或只读 Git 查询，并记录结果 |
| L4 | 平台接入 | 接入 CI/CD、发布平台、流程平台或代码托管平台执行动作；hicode 当前不作为默认能力 |

默认能力：

1. 分析、计划、Review、发布检查默认控制在 L0-L1。
2. 文件修改只能在用户明确要求实现、重构或更新资产时进入 L2。
3. 命令执行必须满足受限命令条件后才能进入 L3。
4. L4 动作不得由 Agent 自动执行，只能建议转既有人工或平台流程。

## 2. 全局禁止操作

所有场景均禁止：

1. 读取或输出密钥、Token、Cookie、Session、连接串、生产账号、生产 IP、内部密钥、`.env`、生产配置或生产凭证。
2. 处理未脱敏客户敏感信息、未脱敏生产数据或生产日志原文。
3. 连接生产环境、调用生产接口、执行生产 SQL、修改生产配置、发布或回滚。
4. 自动提交、自动推送、自动合并、自动发布、自动回滚或替代负责人审批。
5. 删除未确认文件、删除测试、降低断言、跳过 Review 或隐藏风险。
6. 用模板、Hook、门禁建议或模型推断覆盖项目负责人、发布负责人、安全负责人或监管要求。

## 3. 本地修改规则

Agent 进行 L2 本地修改时必须满足：

1. 用户明确要求修改或当前工作包明确要求产出文件。
2. 修改范围可从需求、计划、工作包或用户最新指令中追溯。
3. 不读取敏感配置、密钥、生产数据或未脱敏客户信息。
4. 不扩大到无关重构，不覆盖用户未确认改动。
5. 输出修改文件、行为变化、风险、验证结果或未验证原因。

## 4. 受限命令规则

L3 受限命令必须同时满足：

1. 命令来自目标项目已有脚本、文档、工具配置或用户明确确认。
2. 命令运行在本地非生产环境。
3. 命令不连接生产、不读取密钥、不处理未脱敏生产数据。
4. 命令不执行数据库变更、发布、回滚、提交、推送、合并或删除未确认文件。
5. 输出记录命令、范围、授权来源、是否执行、结果、失败摘要或未执行原因。

允许的受限命令类型：

1. 本地测试、构建、静态检查、格式检查和覆盖率。
2. 敏感信息扫描。
3. 只读 Git 查询。
4. 用户明确要求的本地非生产检查命令。

## 5. 审计证据

涉及计划、Review、测试、发布、门禁、文件修改或命令执行时，输出应尽量记录：

| 证据类型 | 内容 |
|---|---|
| 输入材料清单 | 需求、上下文、diff、报告、测试结果、CI 摘要、人工确认记录等 |
| 检查范围 | 需求编号、分支、Commit/MR、版本、制品、发布申请或节点范围 |
| 结论依据 | 结论、最高风险、关键证据、缺口和不适用说明 |
| 受限命令记录 | 命令、范围、授权来源、结果、失败摘要或未执行原因 |
| 建议动作 | 修复、补证据、人工确认、暂停推进或转平台流程 |
| 上下文更新建议 | 建议更新位置、类型、原因和待确认事项 |

审计证据只说明 Agent 建议如何形成，不代表负责人审批已完成。
