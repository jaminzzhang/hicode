# hicode Rules

## 定位

`references/rules/` 保存 hicode 当前有效的规则依据。它服务根目录 Skill 和专业 Agent 的按需读取，不是默认全量上下文。

规则用于说明如何判断、如何检查、何时停止、如何记录证据和如何输出建议结论。规则不得替代人工审批、发布负责人、生产流程或项目 Owner 决策。

## 目录规划

| 目录 | 用途 |
|---|---|
| `shared/` | 安全红线、金融核心系统风险基线、建议结论、权限边界、Markdown 结构化输出规则 |
| `init/` | 目标项目入口、上下文和项目规则文档初始化规则 |
| `scope/` | 需求澄清、范围界定、编码计划和编码准入规则 |
| `tdd/` | 测试先行、失败复现、受控实现和受限命令规则 |
| `review/` | 代码审查、提交检查、安全/Java/SQL/保险专项检查规则 |
| `release/` | 核心场景测试、发布检查、回滚方案和发布风险规则 |

## 收录原则

1. 只收录当前执行需要的规则、检查口径和结构化输出约束。
2. 旧 Prompt、Gate、Schema、workflow、review-rules 中仍有效的规则应被提炼、去重后进入本目录。
3. 旧 JSON Schema 不作为当前资产保留；稳定字段和枚举以 Markdown 规则维护。
4. guide 类长文不进入本目录，应压缩进对应 Skill 或 Agent。
5. `shared/` 只能放跨场景短规则，避免重新变成大杂烩。

## 禁止事项

1. 不保存输出模板全文；模板属于 `references/templates/`。
2. 不保存历史原文；历史材料属于 `archive/`。
3. 不复制长篇 Prompt、Gate 或 guide 原文。
4. 不输出最终审批、允许合并、允许发布或可以上线。
5. 不放生产命令、生产 SQL、生产配置值、密钥、未脱敏客户信息或未脱敏生产数据。
