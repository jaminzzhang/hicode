# Scope Rules

## 定位

本目录保存需求评审、范围界定、编码计划和编码准入规则。它服务 `hicode:scope`，不直接修改代码。

## 必要输入

1. 需求摘要、PRD、故事链接、会议纪要或脱敏材料。
2. 已有 `docs/PRD_CONTEXT.md`、`docs/PROJ_CONTEXT.md`、`docs/DOMAIN_KNOWLEDGE.md`、`docs/DEFECT_CASES.md`，若缺失需标注缺口。
3. 可能影响的模块、接口、表、配置、外部系统或发布范围。
4. 已澄清问题、未澄清问题和负责人确认记录。

## 停止条件

命中以下情况时，不得建议进入编码：

1. 需求目标、范围、验收标准或核心业务规则不清。
2. P0/P1 风险未关闭，或无法排除保险核心业务、金额、交易一致性、状态、幂等、权限、隐私、监管、发布和回滚风险。
3. 影响范围无法定位到模块、接口、表、配置或外部依赖。
4. 需要 ADR 但未形成决策草案或负责人确认路径。
5. 输入包含未脱敏客户信息、生产数据或密钥。

## 检查维度

| 维度 | 检查规则 |
|---|---|
| 需求完整性 | 目标、范围、验收标准、业务规则、边界、异常和数据来源是否明确 |
| 影响范围 | 系统、模块、接口、类、方法、表、配置、SQL、批处理和外部依赖是否可追溯 |
| 金融核心风险 | 对共享风险基线逐项判断是否涉及、证据是否充分、缺口由谁确认 |
| 测试前置 | 正常、边界、异常、历史缺陷和安全场景是否能转入 TDD |
| ADR 判断 | 架构、数据模型、接口契约、事务边界或发布策略变化是否需要 ADR |
| 上下文沉淀 | 新业务规则、模块事实或缺陷模式只输出更新建议，正式写入需负责人确认 |

## 输出要求

输出应使用 `references/templates/scope/scope-report.md`，并包含：

1. 建议结论。
2. 依据与输入缺口。
3. 需求摘要和范围内/外。
4. 影响范围。
5. 风险清单和阻断建议项。
6. TDD 测试重点。
7. ADR 判断。
8. 待确认问题。
9. 上下文更新建议。
