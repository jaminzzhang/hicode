# Release Rules

## 定位

本目录保存核心场景测试、发布前检查、发布风险、生产验证计划和回滚检查规则。它服务 `hicode:release`，不替代发布负责人、审批流程、CI/CD 或发布平台。

## 核心场景测试规则

1. 发布前必须核对需求、变更范围、领域知识、测试规范、历史缺陷和已有测试证据。
2. 测试建议覆盖必测、回归、边界、异常、历史缺陷和安全场景。
3. 缺少核心场景测试证据时，不得给出无条件发布建议。
4. 测试数据必须脱敏或虚构，不访问真实外部系统、真实数据库或生产环境。

## 发布材料准入

发布检查至少核对：

| 材料 | 检查规则 |
|---|---|
| 发布范围 | 需求清单、模块、分支、制品和版本是否一致 |
| Commit/MR | 是否漏合、错合、多合，Review 是否完成 |
| 测试证据 | 单测、集成、回归、核心场景、CI 和覆盖率是否有结论 |
| 缺陷 | P0/P1 是否关闭，豁免是否有负责人确认 |
| SQL/配置/脚本 | 影响范围、执行顺序、回滚方案、审批和敏感配置是否明确 |
| 接口兼容 | 调用方、渠道、批处理、报表或监管输出是否受影响 |
| 生产验证 | 验证点、负责人、观察窗口和异常处理是否明确 |
| 回滚 | 触发条件、回滚动作、前置条件和负责人是否明确 |

## 不建议发布条件

命中以下任一条件时，输出 `BLOCKED` 或待确认，不得建议发布：

1. 发布范围、需求清单或制品信息缺失。
2. P0/P1 缺陷、Review 问题或测试失败未关闭。
3. 核心业务场景未覆盖或无法排除金融核心系统高风险。
4. SQL、配置、脚本、接口兼容、回滚或生产验证点缺失。
5. 发布材料包含密钥、生产凭证、未脱敏客户信息或生产越权操作。
6. 用户要求 Agent 自动发布、回滚、修改生产配置或读取生产日志。

## 输出要求

输出应使用 `references/templates/release/release-report.md`，并包含：

1. 发布建议结论和最高风险。
2. 发布材料准入检查。
3. 发布范围、需求清单、分支、制品和 Commit/MR 核对。
4. 核心场景测试证据。
5. 测试和缺陷关闭情况。
6. SQL、配置、脚本和接口检查。
7. 生产验证计划、回滚方案和上线后观察建议。
8. 阻断建议、待确认问题和上下文更新建议。
