# 核心场景测试 Skill 示例案例

> 本示例为 P4-WP7 脱敏结构示例，不是真实试点结果，不代表任何真实客户、保单、系统、分支或发布记录。

## 1. 适用 Skill

`references/skills/core-scenario-test/SKILL.md`

## 2. 脱敏输入摘要

需求编号：`REQ-DEMO-2026-001`

需求层证据：批改单生效日期规则调整 PRD 摘要。

变更层证据：示例 diff 摘要显示规则类、错误码、审计事件组装和测试变更。

测试层证据：示例单元测试摘要、接口测试计划和未执行集成测试说明。

缺陷层证据：历史脱敏规则指出“终态保单批改校验缺失”曾导致错误流转。

## 3. 处理重点

1. 需求层和变更层证据是可建议状态硬门槛。
2. 测试层证据不足时可以输出补测建议，但不能写成已覆盖。
3. 场景必须覆盖日期边界、状态流转、审计、异常和回归。
4. 不使用真实客户、真实保单或生产日志。

## 5. 人工复核关注点

1. 需求层和变更层证据是否真实齐备。
2. 测试报告是否覆盖必测场景和回归场景。
3. 未执行测试是否被错误写成已通过。

## 6. 禁止误用说明

1. 不得使用真实客户或真实保单数据构造样例。
2. 不得把“有条件进入发布检查”当作测试负责人最终确认。
3. 不得连接真实外部系统补齐测试证据。
