# 代码审查 Skill 示例案例

> 本示例为 P4-WP7 脱敏结构示例，不是真实试点结果，不代表任何真实客户、保单、系统、分支或发布记录。

## 1. 适用 Skill

`references/skills/code-review/SKILL.md`

## 2. 脱敏输入摘要

审查目标：批改单生效日期规则调整的示例 diff。

固定基准点：`demo-base`

HEAD：`demo-head`

示例 diff 摘要：

1. 新增 `EndorsementEffectiveDateRule` 的回溯日期判断。
2. 新增日期越界错误码。
3. 新增部分单元测试。
4. 未看到终态保单状态限制测试。
5. 未看到审计事件字段校验。

需求来源：

1. `REQ-DEMO-2026-001`
2. 编码计划示例
3. TDD 示例

规范来源：

1. 代码审查 Skill
2. `references/docs/REVIEW_RULES.md`
3. `references/docs/TESTING_GUIDE.md`

## 3. 处理重点

1. 固定审查范围，不凭自然语言摘要审查全部代码。
2. 分开呈现规范轴和需求轴。
3. 严重度必须有依据。
4. 缺少测试或需求证据时不得给无条件通过。

## 5. 人工复核关注点

1. 真实审查必须基于真实 diff 范围和提交列表。
2. P1 问题是否已修复并重新跑测试。
3. 需求轴缺口是否由业务负责人或系统 Owner 确认。

## 6. 禁止误用说明

1. 不得把示例基准点当作真实 commit。
2. 不得在没有 diff 范围时输出无条件通过。
3. 不得把 P1 需求缺失降级为普通风格建议。
